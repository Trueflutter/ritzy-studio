# 04 Design Brief

## Canonical Design Source

The canonical design source is `docs/Vision/05_Brand_and_Design_System.md`.

Implementation agents must not derive style from the exploration HTML files except as secondary visual references. If an exploration file conflicts with the design system markdown, the markdown wins.

## Locked Direction

- Direction: Quiet Gallery
- Type: Cormorant Garamond and DM Sans
- Surface: Bone on Paper
- Accent: Ochre
- Geometry: square corners, 1px hairlines, minimal elevation
- Dark mode: out of scope

## Product Feel

The product is a premium operational design workspace. It must feel editorial, quiet, architectural, precise, and truthful. The room image is the subject; the interface supports it.

## Key Design Rules For Build

- No generic SaaS dashboard styling.
- No rounded blue buttons.
- No purple AI gradients, glows, sparkle icons, or "magic" language.
- No card inside a card.
- No pure black or pure white.
- Use the 4px spacing scale only.
- Use one 52px button height.
- Keep generated concept images visually dominant in review screens.
- Keep product metadata visible and legible.
- Never hide uncertainty behind hover-only UI.
- Always show designer control for substitutions and overrides.

## Screen Families

The MVP contains these screen families:

- project dashboard
- new project flow
- room upload
- design brief
- concept generation and review
- product matching
- product substitution
- final render
- client presentation and export
- settings, if needed

Each screen must follow the per-page UX rules and blueprints in the design system.
